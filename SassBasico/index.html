Sass basico

Sass es CSS con super poderes

Sass es un lenguaje de hojas de estilo que mejora la eficiencia de la escritura y gestión de CSS (Cascading Style Sheets). Su nombre significa "Syntactically Awesome Stylesheets" (Hojas de estilo sintácticamente impresionantes). Con Sass, los desarrolladores pueden utilizar una sintaxis especial que incluye variables, anidamiento de reglas, mixins y funciones para crear hojas de estilo más legibles y mantenibles. Sass no se ejecuta en los navegadores directamente; en su lugar, se compila en CSS estándar antes de su implementación en un sitio web o aplicación. Esto permite a los desarrolladores aprovechar las ventajas de Sass en su proceso de desarrollo, mientras que los usuarios finales solo ven el CSS resultante en sus navegadores. En resumen, Sass es una herramienta que agiliza la escritura de CSS, haciéndola más organizada y flexible, lo que mejora la productividad y calidad del código en proyectos web.

Sass vs Scss

Sass (Syntactically Awesome Stylesheets): Sass originalmente tenía una sintaxis más compacta y basada en indentación, lo que significa que se usaban sangrías en lugar de llaves y puntos y coma. Esto hacía que el código fuera más limpio y legible, pero también requería una estructura estricta. Por ejemplo:

                    
                        .container
                            width: 100%
                            margin: 0 auto

SCSS (Sassy CSS): SCSS, por otro lado, utiliza una sintaxis más similar a CSS estándar, lo que facilita la transición para los desarrolladores que ya están familiarizados con CSS. Las reglas se definen entre llaves y se usan puntos y comas para separar las declaraciones. Por ejemplo:

                    
                        .container {
                            width: 100%;
                            margin: 0 auto;
                        }
